Database

I worked a decade in Microsoft Access, version 97 to 2003. My biggest application is still managing project specifications and sales contacts at Fountain People, Inc.. They say Access doesn't encourage a sound relational approach, but it does if you have ten concurrent users. Building forms in Access introduced me to user interface issues. Access is fun.

A friend asked me to retool his Excel spreadsheet containing data for 10k manufactured items. I normalized the data and moved it into Access. He said he'd like to put it online, and that was my start with MySQL. The database now uses six tables to serve out information like:

  • Manufacter's PN and description
  • After-market supplier's PN and description
  • Close and almost-close substitutes
  • Sub-parts
  • Categories and sub-categories
  • Specific traits by sub-category
  • Images
  • Usage documents like manuals and wiring schematics
